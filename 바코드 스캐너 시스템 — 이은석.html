현장 이해

전달해 주신 자료와 대화 내용을 바탕으로 파악한 현장 환경입니다.

항목내용

취급 상품소형 소품류, 디테일(색상·사이즈·모델) 차이가 미세한 제품 다수

왜 스캔이 필요한가

핵심 문제: 상품 식별에 숙련 기간이 필요합니다.

소품류 수천 종의 디테일 차이를 눈으로 구분하려면 최소 2~3개월의 근무 경험이 필요합니다. 신규 직원이 들어올 때마다 이 학습 비용이 반복됩니다.

해결: 바코드를 스캔하면 쿠팡 상품 이미지와 실사 사진이 즉시 표시됩니다.
첫날 출근한 직원도 상품을 정확히 식별할 수 있습니다.

사용 장면

장면동작효과
포장·검수 포장 전 바코드 스캔 → 이미지로 상품 확인 오배송 방지, 검수 속도 향상
피킹 선반에서 상품 꺼낸 후 스캔 → 맞는 제품인지 확인 오피킹 방지, 동선 내 즉시 확인
신입 교육 모르는 상품 → 스캔으로 즉시 식별 숙련 기간 단축, 교육 비용 절감

전환 비교

현행

  • PC 앞까지 이동하여 조회
  • 로그 상자 클릭 후 USB 스캐너로 스캔
  • xlsx 헤더 불일치 시 인식 실패
  • 포장 위치와 PC 위치가 떨어져 있음

구축 후

  • 헤더 자동 인식, 쿠팡 형식 변경에 유연 대응
  • 피킹 동선 내에서 바로 확인

장비 3개, 연결 2개. 단순하지만 확장에 제약이 없는 구조입니다.

PDA가 NAS에 직접 접근하지 않습니다. 모든 요청은 Mini PC(API 서버)를 거칩니다.
NAS 경로가 변경되더라도 서버 설정만 수정하면 됩니다. PDA 앱은 수정할 필요가 없습니다.

모듈화 설계: 각 기능(조회, 파싱, 이미지 처리)이 독립된 모듈로 구성됩니다. 이후 기능을 추가하거나 기존 사내 프로그램과 연동할 때 기존 구조를 변경하지 않고 모듈 단위로 붙일 수 있는 구조입니다.

데이터 이관

현재 보유하고 계신 데이터를 시스템 DB로 옮기는 작업에 대해 안내드립니다.

이관 대상

파일컬럼 구조처리 방식
coupangmd00_sku_download SKU ID, 상품명, 바코드, 발주가능상태, 담당BM, 발주담당자, 치수, MOQ, 중량 등 17개 컬럼 파서가 헤더를 자동 탐색하여 DB에 적재

헤더 불일치 문제

현행 시스템 로그에 sku_download 헤더 인식 실패 에러가 반복 발생하고 있었습니다.
쿠팡 서플라이어 허브의 내보내기 형식이 변경될 경우 즉시 장애로 이어지는 구조였습니다.

해결 방식은 두 가지 방향이 있습니다. 대표님 판단에 따라 선택해 주시면 됩니다.

방안 A — 자동 매칭

  • 유사한 컬럼명을 자동으로 인식
    예: "SKU ID" / "skuId" / "SKU_ID" → 동일 컬럼
  • 형식이 바뀌어도 중단 없이 처리
  • 매칭 실패 시 로그에 기록

방안 B — 수동 지정

  • 매칭 실패한 컬럼을 로그에 기록
  • 운영자가 직접 컬럼 매핑을 확인하고 수정
  • 의도하지 않은 자동 매칭 오류 방지

대응 가능 파일 형식

형식지원
.xlsx / .xls지원
.csv / .tsv지원 (인코딩 자동 감지, 구분자 자동 판별)
Google Sheets공유 링크로 직접 연동 가능 (API 키 불필요, 공유 설정된 시트 기준)
.ods (LibreOffice)지원
.json지원 (배열 형태의 상품 데이터)
데이터 이관 범위 안내:

기존에 보유하고 계신 SKU, 바코드, 상품명, 이미지 데이터를 이번 시스템 DB로 정리 및 이관하는 작업이 개발 범위에 포함되어 있습니다. 데이터 형식이 일정하지 않은 경우의 정제 작업 역시 개발 범위에 포함됩니다. 별도로 사전 정리하실 필요가 없습니다.
05
신규 데이터 반영

SKU가 계속 증가하는 상황을 전제로 설계합니다.

운영 프로세스

1. 쿠팡 서플라이어 허브에서 최신 xlsx를 다운로드합니다.
2. NAS의 기존 경로(Z:\물류부\scan\)에 저장합니다.
3. 서버가 자동으로 감지하여 DB에 반영합니다.

현재 하고 계신 방식과 동일합니다. 추가로 익혀야 할 절차가 없습니다.

갱신 규칙

상황처리 방식
신규 바코드DB에 추가
기존 바코드, 정보 변경최신 데이터로 갱신
기존 바코드, xlsx에서 누락DB에서 삭제하지 않음 (데이터 보호)

모든 갱신 작업은 로그에 기록됩니다. 몇 건이 추가되었고 몇 건이 수정되었는지 확인하실 수 있습니다.

신규 상품 이미지 연결

자동 매핑:

신규 상품의 이미지 연결은 반자동으로 이루어집니다. NAS의 img/ 또는 real_image/ 폴더에 이미지를 저장하면, 서버가 파일명 규칙(바코드 또는 SKU ID 기반)으로 자동 매칭합니다. 수동으로 codepath.xlsx를 편집할 필요가 없습니다.

codepath.xlsx에 등록된 경로를 우선 참조하며, 등록되지 않은 경우 파일명 규칙 기반으로 자동 탐색합니다.
이미지가 없는 경우에도 앱이 중단되지 않으며, 상품 정보는 정상적으로 표시됩니다.

캐시 구조

계층위치역할효과
서버 캐시 Mini PC NAS에서 읽은 이미지를 PDA 해상도에 맞게 리사이즈 후 저장. 동일 상품 재조회 시 NAS 접근 없이 즉시 응답 이미지 응답 200ms → 20ms
PDA 캐시 PDA 앱 내부 최근 조회한 상품 이미지를 앱 메모리에 보관 (LRU 방식, 최대 500건) 반복 조회 시 네트워크 요청 없이 즉시 표시
DB 캐시 SQLite 바코드 인덱스가 메모리에 상주. 디스크 I/O 없이 조회 DB 조회 30ms → 5ms (재조회)
NAS를 직접 조회하지 않습니다.

모든 이미지는 Mini PC 서버를 경유하며, 한 번 조회된 이미지는 서버에 캐싱되어 이후 요청에서는 NAS에 접근하지 않습니다.

08
운영 안정성

시스템이 정상 작동하지 않을 때 어떤 일이 발생하는지, 그리고 어떻게 복구되는지에 대한 설계입니다.

장애 상황별 대응

상황증상대응
Mini PC 꺼짐 PDA에서 조회 불가 PDA 화면에 "서버 연결 안 됨" 안내 표시. Mini PC 전원 켜면 서버 자동 시작
NAS 접근 불가 이미지 미표시 상품 정보(상품명, SKU 등)는 정상 표시. 이미지 영역에 "이미지 로드 실패" 표시
WiFi 끊김 PDA 조회 불가 재연결 시 자동 복구. 별도 앱 재시작 불필요
xlsx 형식 오류 DB 갱신 실패 기존 데이터 유지 (덮어쓰지 않음). 실패 원인을 로그에 기록
설계 원칙: 어느 한 부분이 실패해도 전체 시스템이 중단되지 않습니다.
NAS가 꺼져도 상품 정보는 조회되고, xlsx 파싱이 실패해도 기존 데이터는 보존됩니다.
