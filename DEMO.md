## 2. 현장 도착 후 5분 체크

1. PDA 전원 → 앱 자동 실행 확인 (스플래시 → 메인)
2. 상단 검색창에 "테스트" 치고 엔터 → 상품 리스트 뜨는지
3. 미리 준비한 바코드 1개 스캔 → 0.3~0.5초 내 결과 표시
4. 서버 연결 실패 시: 설정 화면 들어가서 서버 주소 재입력 후 "연결 테스트"

### ⭐ 물류PC 라벨 에이전트 복구 (반드시)

현재 물류PC의 `print_agent.py`가 꺼져있음. 라벨 시연 전에 반드시 살려야 함.

**순서 (2분)**:

1. 물류PC 자리로 이동 → "프린터 상태 잠깐 확인할게요" 멘트
2. Win키 → `cmd` 엔터 → 아래 한 줄 실행:
   ```
   pythonw C:\print_agent.py
   ```
3. 창 닫고 돌아와서, 노트북이나 PDA 브라우저에서 확인:
   ```
   curl http://100.123.11.122:7777/health
   ```
   `200 OK` + JSON 응답 뜨면 성공

**만약 pythonw 안 먹으면**:

- `C:\Python*\pythonw.exe C:\print_agent.py`
- 또는 `python C:\print_agent.py` (창 뜨는데 상관없음)

**가는 김에 이후 자동화를 위해 (선택)**:
레지스트리에 자동실행 등록이 풀려있을 수 있음. 재부팅 후에도 뜨도록:

```
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v PrintAgent /t REG_SZ /d "pythonw.exe C:\print_agent.py" /f
```

## 3. 시연 순서 (10~15분)

### ① 바코드 스캔 (핵심)

- 실제 상품 바코드 스캔
- 결과: 상품명 / 이미지 / 위치 (몇구역 몇번째 셀) 즉시 표시
- **말 포인트**: "스캔하자마자 0.5초 안에 어디 있는지 보여줍니다"

### ② 도면에서 위치 확인

- 스캔 결과에서 "위치 보기" → 도면 팝업
- 해당 셀이 노란색 하이라이트
- 핀치줌 시연 (두 손가락으로 벌려 확대)
- **말 포인트**: "도면 편집은 어드민에서 하고, PDA에서는 보기만 합니다"

### ③ 셀 상세 (v5.4.5 핵심 변경)

- 도면에서 아무 셀 탭 → 셀 상세 진입
- 셀 사진 풀스크린 표시, 하단에 "OO구역 A-1" 라벨
- 하단 네비 3버튼: 도면 / (라벨) / 편집
- "편집" → "사진 변경" 확인 → 카메라/갤러리 선택
- **말 포인트**: "이전에 층 개념 있던 거 없애고 셀 하나에 사진 하나로 단순화했어요"

### ④ 박스 상세

- 특정 상품에 연결된 박스가 있다면 박스 상세 진입
- 박스 사진 + 담긴 상품 리스트

### ⑤ 상품 배치 (필요 시)

- 메인에서 상품 검색 → 상품 선택 → "배치" → 셀 선택 → 확인

### ⑥ 설정 화면 (마지막)

- 서버 주소 / API 키 / 연결 테스트 버튼
- 앱 버전 5.4.5 확인

## 6. 돌발 상황 대응

### 스캔이 안 될 때

1. DataWedge 프로파일 확인 (설정 → DataWedge → scan.warehouse)
2. 앱 강제종료 → 재실행
3. PDA 재부팅 (최후)

### 라벨 인쇄 "물류PC 응답 없음" 떴을 때

- 2번 섹션의 `pythonw C:\print_agent.py` 다시 실행 (죽은 것)
- 프린터 전원/USB 확인
- 라벨 템플릿 `.btw` 파일 있는지 확인

### 사진 업로드 실패

- WiFi 신호 확인
- 서버 디스크 여유 확인 (노트북으로 SSH)

### 앱 크래시

- logcat 수집 준비: `adb logcat -d | grep scan.warehouse` (노트북에서)
- 즉시 "확인해보겠습니다" 멘트 후 재현 조건 메모

### 서버 다운

- 노트북에서 `ssh mini` → `systemctl restart scan-server` 또는 수동 `uv run uvicorn app.main:app`
