## 3. 아키텍처 요약

```
PDA (TC60) --WiFi--> Mini PC (서버) --WebDAV--> NAS (이미지/데이터)
```

- PDA 앱: 바코드 스캔, 상품 검색, 이미지 표시. 오프라인 캐시(Room DB).
- Mini PC 서버: FastAPI + SQLite. xlsx 파싱, 바코드 조회 API, NAS 이미지 프록시. systemd 자동 시작. 매일 자동 백업.
- NAS: 쿠팡 썸네일(img/), 실사(real_image/), xlsx 데이터 저장소. WebDAV로 서버 접근.

데이터 흐름: NAS에 xlsx 업로드 -> 서버 자동 감지/파싱 -> DB 적재 -> PDA에서 바코드 스캔 -> API 조회 -> 이미지 반환

## 5. 클라이언트가 물어볼 수 있는 질문 & 답변

**"진행 상황이 어떤가요?"**
-> 8개 마일스톤 중 7개 완료, 85% 진행. 남은 건 PDA 실기기 테스트와 현장 설치인데, 이건 기기/접속 정보를 받아야 진행 가능한 부분입니다.

**"PDA 없이 어떻게 개발하나요?"**
-> adb 명령어로 바코드 스캔을 시뮬레이션합니다. DataWedge Intent API 방식이라 실기기 없이도 앱 로직 검증이 가능합니다. 실기기 도착하면 실제 스캔 테스트만 하면 됩니다.

**"기존 프로그램이랑 뭐가 다른가요?"**
-> 기존: PC에서만 사용, USB 스캐너 연결 필요. 신규: PDA 들고 다니면서 스캔, 이미지를 크게 바로 표시, 데이터 자동 갱신. 기존 NAS 폴더 구조와 xlsx 파일은 그대로 사용합니다.

**"데이터 이관은 어떻게?"**
-> 기존에 쓰시던 codepath.xlsx, sku_download.xlsx 그대로 사용합니다. NAS에 올려두시면 서버가 자동으로 읽어서 반영합니다. 별도 변환 작업 없습니다.

**"인터넷 필요한가요?"**
-> 외부 인터넷은 필요 없습니다. 창고 내부 WiFi만 있으면 됩니다. PDA와 Mini PC가 같은 WiFi에 연결되어 있으면 동작합니다.

**"보안은?"**
-> 내부망 전용이라 외부에서 접근 불가합니다. 이미지 경로 조작 공격 차단, 데이터 암호화 적용했습니다.

**"백업은?"**
-> 매일 새벽 3시 자동 백업, 최근 7일치 보관합니다. 문제 발생 시 명령어 한 줄로 복구 가능합니다.

**"추가 개발은?"**
-> Phase 1 완료 후 사용해보시고, 필요하신 기능이 있으시면 말씀해주세요. (임의 제안 하지 않는다)

## 7. 클라이언트 소통 시 주의사항

- 기획 주도권은 클라이언트에게 있다. 기능 제안하지 않는다.
- AI가 쓴 느낌의 장문/과잉 설명 하지 않는다. "별로입니다"라고 직접 말씀하신 적 있음.
- 기존 프로그램(scan13.exe)을 "초보적", "구식" 등으로 비하하지 않는다. 이전에 지적받음.
- Phase 2, 3 언급하지 않는다. 클라이언트가 직접 삭제 요청했음. Phase 1만.
- "확장 가능합니다", "향후 대비" 류의 표현 최소화. 현재 동작하는 결과물로 보여준다.
- 결과물로 증명. "말이 필요 없게 만들어주신다면 당연히 합니다"라고 하신 분.
