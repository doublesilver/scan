# 현장 작동 시나리오

> 대상: 물류창고 작업자 (5층, 층당 70~100평)
> 기기: Zebra TC60 PDA / 버전: v5.1.0

---

## 시나리오 1 — 최초 앱 설치 후 서버 연결

**상황:** 새 PDA에 APK를 설치하고 처음 앱을 켰다.

1. 앱 실행 → 스플래시 화면 잠깐 표시 후 메인 화면 진입
2. 우측 상단 ⚙️ 설정 버튼 터치
3. 서버 주소 입력란에 Mini PC IP 입력 (예: `http://192.168.1.100:8000`)
4. [저장] → "연결 확인 중..." → "서버 연결됨" 토스트 표시
5. 뒤로가기 → 메인 화면 "PDA 버튼을 눌러 스캔하세요" 대기 상태

> **자동 발견 옵션:** IP를 모를 경우 [자동 발견] 버튼 터치 → 사내 WiFi 서브넷 자동 스캔 → 서버 감지 시 IP 자동 입력

---

## 시나리오 2 — 바코드 스캔으로 상품 확인 (가장 기본 작업)

**상황:** 선반에서 상품을 집다가 "이게 맞는 상품인지" 확인하고 싶다.

1. PDA 측면 스캔 트리거 버튼 누름
2. 상품 바코드(EAN-13) 스캔
3. 0.3~0.5초 내 결과 표시:
   - 상품 이미지 (화면 상단 2/3 채움)
   - 상품명 ("폰케이스 투명 아이폰15" — 스페이스쉴드 자동 제거됨)
   - 바코드 (`8809461` \*\*`70008`` — 끝 5자리 볼드)
   - SKU ID
4. 이미지가 쿠팡 썸네일이라 실물과 다를 경우 → 이미지 터치 → 실사 이미지로 전환
5. 다시 터치 → 썸네일로 복귀
6. 상품 카드 전체 터치 → 상세 화면 진입 (바코드 전체 목록, 위치 정보 등 확인)
7. 뒤로가기 → 메인 대기 화면

---

## 시나리오 3 — 상품명으로 검색

**상황:** 바코드가 훼손돼서 스캔이 안 된다. 상품명으로 찾아야 한다.

1. 상단 검색창 터치 → 키보드 올라옴
2. "게임 컨트롤러" 입력 → [검색] 버튼 또는 키보드 검색 키
3. 목록에 20건 이내 결과 표시
4. 원하는 상품 터치 → 바코드로 스캔한 것과 동일한 결과 화면 표시
5. 검색창 우측 X 버튼 → 대기 화면으로 복귀

---

## 시나리오 4 — 상세 화면에서 이미지 슬라이드 확인

**상황:** 같은 SKU에 색상 변형이 여러 개라 이미지를 여러 장 봐야 한다.

1. 스캔 결과 카드 터치 → 상세 화면
2. 상단 이미지 영역에서 좌우 스와이프 → 다음 이미지로 이동 (ViewPager2)
3. 썸네일 이미지에서 실사 이미지로 토글하며 비교
4. 하단에 바코드 목록 전체 표시 — 해당 SKU에 묶인 모든 바코드 확인 가능
5. 뒤로가기

---

## 시나리오 5 — 도면에서 상품 위치 확인

**상황:** "이 상품이 몇 층 창고 어디 있더라?"

1. 상품 스캔 → 결과 화면
2. 상세 화면 진입 → 위치 정보 확인 (예: "A구역 A-5")
3. 위치 정보 터치 또는 도면 아이콘 → WarehouseMapDialog 팝업
4. 전체 창고 도면에서 해당 셀 주황색 하이라이트 표시
5. 다이얼로그 닫기

---

## 시나리오 6 — 셀에 상품 배치 등록

**상황:** 새로 입고된 박스를 선반에 정리하면서 어느 셀에 뭐가 있는지 기록해야 한다.

1. 메인 화면에서 도면 버튼 터치 → WarehouseMapDialog
2. 정리 중인 선반 셀 터치 (예: B구역 2행 3열)
3. CellDetailActivity 진입
   - 상단: 셀 위치 정보 (B구역, 2-3)
   - 층 탭: 하단(1층) / 중단(2층) / 상단(3층)
4. 비어있는 셀이면 자동으로 편집모드 진입
5. [+ 상품 추가] 버튼 터치
6. PDA 트리거로 상품 바코드 스캔 → 상품 자동 매칭
7. 상품 카드가 해당 층에 등록됨
8. 카메라 아이콘 터치 → 현장 사진 촬영 → NAS 자동 업로드
9. ← → 버튼으로 옆 셀(A구역 2-4)로 이동해 동일 작업 반복
10. 뒤로가기

---

## 시나리오 7 — 셀 상품 정보 수정

**상황:** 선반 정리 후 잘못 등록된 상품을 빼야 한다.

1. 도면 → 해당 셀 터치 → CellDetailActivity
2. 편집 모드 진입 (우측 상단 편집 버튼 터치)
3. 잘못 등록된 상품 카드를 롱프레스 → "삭제하시겠습니까?" 다이얼로그
4. [확인] → 상품 제거됨
5. 올바른 상품 바코드 스캔 → 재등록

---

## 시나리오 8 — 외박스 QR 스캔

**상황:** 외박스에 붙은 QR코드가 있는데 이 박스 안에 뭐가 들었는지 확인하고 싶다.

1. 검색창에 PDA 트리거로 박스 QR 스캔 (BOX-로 시작하는 QR)
2. BoxDetailActivity 진입:
   - 박스명, 위치 표시
   - 6블록 그리드 (박스 내 구획)
   - 인라인 도면 미니맵 (이 박스가 창고 어디 있는지)
3. 블록 터치 → 해당 구획 상품 상세 표시 또는 외부 링크(쿠팡 등) 열기
4. 뒤로가기

**미등록 QR인 경우:**

1. 스캔 → "등록되지 않은 박스입니다. 등록하시겠습니까?" 다이얼로그
2. [등록하기] → BoxRegisterActivity → 박스명, 위치, 멤버 상품 입력 후 저장

---

## 시나리오 9 — 장바구니로 발주 목록 만들기

**상황:** 재고가 부족한 상품들을 돌아다니며 스캔해서 발주 목록을 만들어야 한다.

1. 상품 스캔 → 결과 화면
2. 하단바 [장바구니] 버튼 터치 → "장바구니 추가 중..." → "추가됨" 토스트
3. 다음 상품으로 이동 → 스캔 → [장바구니] 반복
4. 작업 완료 후 담당자가 Mini PC에서 구글시트 열면 수집된 목록 자동 반영됨

---

## 시나리오 10 — WiFi 음영지역에서 스캔 (오프라인 상태)

**상황:** 창고 내 WiFi 신호가 약한 구석에서 작업 중이다.

1. 상단에 "오프라인 모드" 배너 표시됨
2. 이전에 스캔한 적 있는 바코드 스캔 → Room DB 캐시에서 즉시 결과 표시
3. 처음 스캔하는 상품 → "서버에 연결할 수 없습니다. 재시도" Snackbar 표시
4. [재시도] 터치 또는 WiFi 위치로 이동 후 자동 재시도 (RetryInterceptor 3회)
5. 서버 연결 복구 → 오프라인 배너 사라짐 → 정상 스캔 재개

---

## 시나리오 11 — 앱 업데이트

**상황:** 관리자가 서버에 새 버전 APK를 올려뒀다.

1. 앱 실행 → 스플래시에서 서버 버전 자동 체크
2. 현재 앱보다 새 버전 감지 → "새 버전이 있습니다 (v5.2.0). 업데이트하시겠습니까?" 다이얼로그
3. [업데이트] → APK 자동 다운로드 → 설치 유도
4. 설치 완료 후 앱 재실행

---

## 시나리오 12 — 라벨 인쇄

**상황:** 바코드 없는 상품에 자체 바코드 라벨을 부착해야 한다.

1. 상품 검색 또는 스캔 → 상세 화면
2. 인쇄 버튼 터치
3. TSC TE10 프린터로 바코드 라벨 자동 출력
4. 출력된 라벨을 상품에 부착

---

## 시나리오 13 — 웹 에디터로 도면 수정 (관리자)

**상황:** 창고 레이아웃이 바뀌어서 도면을 수정해야 한다. PDA보다 PC에서 하는 게 편하다.

1. PC 브라우저에서 `http://[MiniPC IP]:8000/admin/map-editor` 접속
2. 도면 에디터 열림 — 셀 드래그 앤 드롭으로 위치 조정
3. 셀 라벨 더블클릭 → 이름 수정
4. 신규 셀 추가, 불필요한 셀 삭제
5. [저장] → 서버에 즉시 반영
6. PDA에서 도면 열면 수정된 레이아웃으로 보임

---

## 오류 상황 처리

| 상황                 | 앱 동작                                      |
| -------------------- | -------------------------------------------- |
| 미등록 바코드 스캔   | "등록된 상품이 없습니다" Snackbar            |
| 서버 연결 실패       | "서버에 연결할 수 없습니다" + [재시도] 버튼  |
| 이미지 로딩 실패     | 플레이스홀더 이미지 표시 (회색 박스)         |
| 외박스 미등록 QR     | "등록하시겠습니까?" 다이얼로그               |
| 카메라 권한 없음     | "카메라 권한이 필요합니다" 안내 후 권한 요청 |
| 중복 스캔 (300ms 내) | debounce로 첫 번째 스캔만 처리               |
