01
프로젝트 개요

쵸미앤세븐·스페이스쉴드 브랜드의 워치 스트랩 제품 데이터를 구조화된 전용 시스템으로 전환하는 프로젝트입니다.

현재 상황

21 구글시트 탭 수
596 총 제품 수
29~36 탭별 컬럼 수
107 제품 패밀리 수

핵심 문제

현재 — 구글시트 21탭

  • 기종 추가 시 컬럼이 늘어나는 구조. 이미 29~36컬럼
  • 탭마다 같은 제품이 중복 기재. 수정 시 전부 반영해야 함
  • 런칭 현황을 한눈에 볼 수 없음. 탭을 오가며 확인
  • 이미지 경로와 제품 데이터가 분리되어 연결 불가
  • 기획 단계 제품과 판매 중인 제품이 뒤섞임

구축 후 — 스트랩 DB

  • 제품 1건 수정하면 전체 뷰에 즉시 반영
  • 기종 추가는 호환 모델 테이블에 행 1줄 추가로 끝
  • 어떤 기종에 어떤 제품이 있는지 교차표로 즉시 확인
  • 이미지 509개 연결 완료. 제품 카드에 바로 표시
  • 상태별(기획/샘플링/입고/판매/단종) 분리 관리

무엇을 만드는가

제품 조회 도구이면서 동시에 신제품 기획 도구입니다. "이 기종에 어떤 스트랩을 아직 안 냈는지", "이 디자인이 어떤 기종으로 파생됐는지"를 즉시 확인할 수 있는 스트랩 백과사전입니다.

03
핵심 기능 4가지
VIEW 01

제품 카탈로그

전체 제품을 검색·필터·카드 뷰로 조회합니다.

  • 제품명 실시간 검색
  • 카테고리 / 재질 / 체결방식 다중 필터
  • 카드 뷰 ↔ 테이블 뷰 전환
  • 제품 클릭 시 상세 모달 (이미지 + 속성 + 호환기종 + 가격)
VIEW 02

런칭 매트릭스

107개 디자인 × 14개 기종 교차표.

  • "이 기종에 뭘 출시했고 뭘 안 했는지" 한눈에
  • 갤럭시워치9 등 신기종 대응 시 공백 즉시 확인
  • 미출시 조합만 빠르게 필터 가능
  • 신제품 기획 시트로 활용
VIEW 03

제품 계보

107개 패밀리의 기종별 파생 관계를 표시합니다.

  • 하나의 디자인이 어떤 기종으로 파생됐는지 시각화
  • 네이밍 중복 방지. 유사 디자인 재출시 여부 판단
  • 스트랩 백과사전으로 활용
  • 단종·파생 이력 추적
VIEW 04

파이프라인

기획부터 단종까지 상태별 칸반 보드.

  • 기획중 / 샘플링 / 일부입고 / 판매중 / 단종
  • 공장 제안 제품 관리
  • 프레스티지 라인업 추적
  • 입고 예정 제품 현황 파악

관리 기능 (CRUD)

4개 뷰는 조회 전용이 아닙니다. 각 뷰에서 직접 수정·추가·삭제가 가능합니다. 제품 상태 변경, 새 기종 추가, 가격 수정을 화면 안에서 처리합니다.

05
데이터 구조

기존 vs 신규 구조

현재 — 구글시트 21탭

  • 탭마다 독립적으로 존재. 탭 간 연결 없음
  • 컬럼 29~36개. 기종 추가 시 컬럼 추가
  • 제품 상태, 이미지, 가격이 같은 행에 혼재
  • 패밀리 개념 없음. 파생 관계 파악 불가

신규 — 정규화 DB

  • 테이블 분리 후 관계로 연결. 중복 없음
  • 기종 추가는 행 1개 추가. 컬럼 구조 불변
  • 제품 / 속성 / SKU / 이미지 / 가격 / 상태 분리
  • 패밀리 107개 정의. 파생 관계 추적 가능
