# 📋 물류창고 도면 아키텍처 — 인터뷰 스펙 v2

> **한눈에 보기**
>
> - **목표:** 도면(창고 배치도)을 기준으로 상품 위치를 관리하는 시스템 구축
> - **변경:** 하나의 큰 덩어리 데이터 → 구역/셀/층/상품으로 깔끔하게 분리
> - **등록 단위:** 개별 SKU(제품)가 아니라 상품군(상품 마스터) 단위
> - **사용 기기:** PDA(Galaxy S20+ / Zebra TC60)가 1순위, 웹은 보조
> - **효과:** 창고 직원이 PDA로 바코드만 찍으면 도면에 자동 반영

## 🎯 목표

도면의 각 칸(셀)을 **상품 위치의 유일한 기준점**으로 삼고,
데이터베이스(DB)를 깔끔하게 분리하여 **창고가 늘어나도 문제없는 구조**를 만든다.

```
현재: 도면 데이터가 하나의 큰 덩어리(JSON)에 뭉쳐 있음
  ↓
변경: 구역 / 셀 / 층 / 상품을 각각 별도 테이블로 분리
  ↓
결과: 개별 수정이 쉬워지고, 데이터가 꼬이지 않음
```

---

## 🗂️ 데이터 구조 (전체 그림)

아래는 데이터가 어떤 순서로 연결되는지 보여주는 구조도입니다.

```
상품 마스터 (상품군)
├── 이름, 대표이미지, 카테고리
└── 제품(SKU) 여러 개
    └── 바코드, 상품명, 위치 정보

창고 구역 (예: 501호, 포장다이)
├── 코드, 이름, 행/열 수
└── 셀 (도면의 한 칸)
    ├── 행, 열, 라벨, 상태, 배경색
    └── 층 (상단/중단/하단)
        └── 층별 상품 (한 층에 여러 상품 가능)
            └── 상품 마스터 연결, 사진, 메모
```

## 💡 핵심 원칙

### 1. 도면 셀 = SSOT(모든 데이터의 기준이 되는 단일 출처)

> 비유: **주소록이 하나만 있으면 혼란 없음.**
> 전화번호를 수첩, 폰, 컴퓨터에 따로 적으면 어디가 맞는지 헷갈리듯,
> 상품 위치도 "도면"이라는 한 곳에서만 관리하면 혼란이 없습니다.

- 상품 위치는 도면에서만 편집합니다.
- 저장하면 상품 정보 쪽에도 자동으로 반영됩니다.

### 2. 자동 동기화

> 비유: **도면에 등록하면 상품 정보에도 자동 반영.**
> 구글 캘린더에 일정 넣으면 폰에서도 바로 보이는 것과 같습니다.

- 도면 셀에 상품을 등록하면, 해당 상품의 모든 SKU(제품) 위치가 자동 업데이트됩니다.
- 도면에서 상품을 제거하면, 위치 정보도 자동 초기화됩니다.

### 3. 상품(마스터) 단위 등록

- 셀 층에는 **상품군**(예: "매트 그레인 스트랩")을 등록합니다.
- 해당 상품군에 속한 개별 SKU(색상/사이즈별 제품)는 자동으로 따라갑니다.

### 4. 한 층 = 여러 상품

- 하나의 선반 층에 여러 종류의 상품이 함께 놓일 수 있습니다.

### 5. PDA 앱 = 1순위

- 구역 추가/삭제, 행열 변경, 셀 내용 편집 모두 PDA에서 가능합니다.
- 창고 직원이 현장에서 바로 작업할 수 있습니다.

### 6. 웹 에디터 = 보조 도구

- PDA 앱과 디자인을 통일합니다.
- 사무실에서 초기 세팅하거나 전체 현황을 볼 때 사용합니다.

### 7. 확장 대비

- 창고 추가, 셀 200개 이상, 동시 사용자 증가에 대응할 수 있는 구조입니다.
