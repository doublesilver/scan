# 🏗️ 도면 아키텍처 세부 설계서

## 📋 한눈에 보기

| 항목              | 내용                                                                       |
| ----------------- | -------------------------------------------------------------------------- |
| **뭘 바꾸나**     | 도면 데이터를 하나의 덩어리에서 구역/셀/층/상품으로 분리                   |
| **왜 바꾸나**     | 셀 하나만 수정하려 해도 전체를 교체해야 하는 비효율 제거                   |
| **뭐가 좋아지나** | PDA에서 바코드 찍으면 바로 등록, 위치 자동 반영, 구역 편집도 현장에서 가능 |

```
[현재 문제]                          [변경 후]
도면 전체가 한 덩어리         →     구역/셀/층/상품을 개별 관리
셀 하나 고치려면 전체 교체    →     해당 셀만 수정
상품 위치를 수동 입력          →     도면 등록하면 자동 반영
구역 편집은 웹에서만           →     PDA에서도 구역 추가/삭제 가능
```

## 🏗️ 데이터 구조 변경

### Before: 현재 방식

현재는 도면 정보가 하나의 큰 JSON(텍스트 덩어리)에 모두 담겨 있습니다.

```
┌─────────────────────────────────────────┐
│  map_layout (하나의 큰 덩어리)            │
│                                         │
│  { zones: [                             │
│      { name: "501호",                   │
│        cells: [                         │
│          { row: 1, col: 1,              │
│            levels: [                    │
│              { items: ["스트랩A"] }      │
│            ]                            │
│          },                             │
│          ... 모든 셀 데이터 ...           │
│        ]                                │
│      },                                 │
│      ... 모든 구역 데이터 ...             │
│    ]                                    │
│  }                                      │
│                                         │
│  → 셀 하나 고치려면 이 전체를 읽고,       │
│    수정하고, 전체를 다시 저장해야 함       │
└─────────────────────────────────────────┘
```

### After: 새 방식

구역, 셀, 층, 상품을 각각 독립된 테이블(서류함)로 분리합니다.

```
┌──────────────┐
│  창고 구역     │  예: 501호, 포장다이, 502호
│  (서류함 1)    │  이름, 행/열 수, 순서
└──────┬───────┘
       │ 하위에
       ▼
┌──────────────┐
│  셀 (칸)      │  예: 1-1, 1-2, 1-3 ...
│  (서류함 2)    │  행, 열, 라벨, 상태, 색상
└──────┬───────┘
       │ 하위에
       ▼
┌──────────────┐
│  층           │  예: 상단(3층), 중단(2층), 하단(1층)
│  (서류함 3)    │  층 번호, 라벨
└──────┬───────┘
       │ 하위에
       ▼
┌──────────────┐
│  층별 상품     │  예: 매트 그레인 스트랩, 밀레니즈 스트랩
│  (서류함 4)    │  상품 마스터 연결, 사진, 메모
└──────────────┘
```

> **비유:** 서류를 하나의 큰 봉투에 다 넣어두면 하나 찾으려 해도 전부 꺼내야 합니다.
> 서류함을 칸별로 나누면 원하는 칸만 열어서 바로 꺼낼 수 있습니다.
> 이번 변경이 바로 그 "서류함 정리"입니다.

### 기존 데이터는 자동 변환됩니다

```
기존 JSON 덩어리
    ↓  자동 마이그레이션 (프로그램이 알아서 변환)
    ↓
구역 테이블에 INSERT ← zones 데이터
셀 테이블에 INSERT   ← cells 데이터
층 테이블에 INSERT   ← levels 데이터
상품 테이블에 INSERT  ← 상품 매칭 데이터
    ↓
기존 데이터 유실 없음 ✓
```

<summary>🔧 개발자용: DB 스키마 (CREATE TABLE문) — 클릭하여 펼치기</summary>

### 새 테이블

```sql
-- 구역
CREATE TABLE warehouse_zone (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    rows INTEGER NOT NULL DEFAULT 3,
    cols INTEGER NOT NULL DEFAULT 4,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);
CREATE INDEX idx_zone_code ON warehouse_zone(code);

-- 셀
CREATE TABLE warehouse_cell (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    zone_id INTEGER NOT NULL REFERENCES warehouse_zone(id) ON DELETE CASCADE,
    row INTEGER NOT NULL,
    col INTEGER NOT NULL,
    label TEXT DEFAULT '',
    status TEXT DEFAULT 'empty',
    bg_color TEXT DEFAULT '',
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now')),
    UNIQUE(zone_id, row, col)
);
CREATE INDEX idx_cell_zone ON warehouse_cell(zone_id);

-- 층
CREATE TABLE cell_level (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cell_id INTEGER NOT NULL REFERENCES warehouse_cell(id) ON DELETE CASCADE,
    level_index INTEGER NOT NULL DEFAULT 0,
    label TEXT DEFAULT '',
    created_at TEXT DEFAULT (datetime('now')),
    UNIQUE(cell_id, level_index)
);
CREATE INDEX idx_level_cell ON cell_level(cell_id);

-- 층별 상품 (한 층에 여러 상품 가능)
CREATE TABLE cell_level_product (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    level_id INTEGER NOT NULL REFERENCES cell_level(id) ON DELETE CASCADE,
    product_master_id INTEGER REFERENCES product_master(id),
    photo TEXT DEFAULT '',
    memo TEXT DEFAULT '',
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);
CREATE INDEX idx_lp_level ON cell_level_product(level_id);
CREATE INDEX idx_lp_master ON cell_level_product(product_master_id);
```

### product_master 확장 (기존 테이블에 컬럼 추가)

```sql
ALTER TABLE product_master ADD COLUMN category TEXT DEFAULT '';
ALTER TABLE product_master ADD COLUMN representative_image TEXT DEFAULT '';
```

### 마이그레이션 전략

1. 위 CREATE TABLE문을 `schema.py` MIGRATIONS[9]에 추가
2. Python 마이그레이션 함수 `migrate_json_to_tables(db)` 실행
   - 기존 `map_layout.data` JSON 읽기
   - zones → `warehouse_zone` INSERT
   - cells → `warehouse_cell` INSERT
   - levels → `cell_level` INSERT
   - 상품 → `cell_level_product` INSERT (product_master 자동 매칭)
3. 웹 에디터 호환: `GET /api/map-layout`은 새 테이블에서 기존 JSON 형태로 조립하여 반환

### 📱 구역 편집 다이얼로그

구역 이름이나 행/열 수를 변경할 때 나타나는 팝업입니다.

```
┌──────────────────────────────────┐
│          구역 설정                │
│                                  │
│  이름:  ┌──────────────────┐     │
│         │ 501호            │     │
│         └──────────────────┘     │
│                                  │
│  코드:  ┌──────────────────┐     │
│         │ 1                │     │
│         └──────────────────┘     │
│                                  │
│  행:    [  3  ]   [−]  [+]      │ ← +/- 버튼으로 조절
│                                  │
│  열:    [  4  ]   [−]  [+]      │
│                                  │
│                                  │
│  [🗑️ 삭제]       [취소]  [저장]  │
└──────────────────────────────────┘

⚠️ 주의사항:
   행/열을 늘리면 → 새 셀이 자동 생성됨
   행/열을 줄이면 → 경고 표시 후, 확인하면 해당 셀 삭제
   기존 셀의 데이터는 그대로 보존됨
```

### 📱 상품 등록 흐름 (단계별)

PDA에서 상품을 셀에 등록하는 전체 과정입니다.

```
① [+ 상품 추가] 버튼 터치
   │
   ▼
②  ┌─────────────────────────┐
   │  등록 방법 선택           │
   │                          │
   │  [📷 바코드 스캔]         │ ← PDA 하드웨어 스캐너
   │  [🔍 상품 검색]           │ ← 이름으로 검색
   └─────────────────────────┘
   │
   ▼
③ 바코드 스캔 또는 검색
   │
   ▼
④ 서버에서 자동 매칭
   바코드 → 제품(SKU) 찾기 → 상품 마스터 찾기
   │
   ▼
⑤ 셀의 해당 층에 상품 등록 완료
   │
   ▼
⑥ 자동 동기화 실행
   해당 상품의 모든 SKU 위치 정보가 자동 업데이트됨
   │
   ▼
⑦ 화면 갱신
   상품 카드가 추가된 것이 바로 보임
```

## 🔗 위치 동기화

도면과 상품 정보가 항상 일치하도록 자동으로 맞춰주는 기능입니다.

### 도면에서 등록 → 상품에 자동 반영

```
[도면에서 상품 등록]
       │
       ▼
셀에 상품 마스터 연결
       │
       ▼
해당 상품 마스터의 모든 SKU(제품) 조회
       │
       ▼
각 SKU의 위치 정보 자동 업데이트
  예: "1구역 1-3"
       │
       ▼
✅ 상품 목록에서도 위치가 표시됨


[도면에서 상품 제거]
       │
       ▼
해당 상품 마스터의 SKU 위치를 초기화
  (단, 다른 셀에도 같은 상품이 있으면 그쪽 위치 유지)
       │
       ▼
✅ 위치 정보 정리 완료
```

### 상품 스캔 → 도면에서 위치 표시

```
[PDA로 바코드 스캔]
       │
       ▼
바코드 → 제품(SKU) → 위치 정보 조회
       │
       ▼
도면 화면에서 해당 셀 하이라이트 표시
       │
       ▼
✅ "이 상품은 1구역 1-3에 있습니다" 표시
```

## 🔌 서버에서 할 수 있는 조작 목록

서버 API(앱과 서버가 소통하는 통로)를 통해 아래 작업을 수행할 수 있습니다.

### 구역 관리

- **구역 목록 보기** — 전체 구역과 각 구역의 셀 개수 확인
- **구역 추가** — 이름, 코드, 행/열 수를 지정하여 새 구역 생성
- **구역 수정** — 이름, 행/열 수, 코드 변경
- **구역 삭제** — 구역과 하위 셀/층/상품 모두 삭제

### 셀 관리

- **셀 목록 보기** — 구역 내 모든 셀과 층/상품 정보를 한번에 조회
- **셀 상세 보기** — 특정 셀의 층별 상품 상세 정보
- **셀 수정** — 라벨, 상태, 배경색 변경

### 층 관리

- **층 추가** — 셀에 새로운 층 추가
- **층 삭제** — 층과 하위 상품 모두 삭제

### 상품 관리

- **상품 등록** — 바코드/SKU/상품마스터 ID로 자동 매칭하여 등록
- **상품 제거** — 셀에서 상품 연결 해제
- **사진 업로드** — 상품 사진 첨부
- **사진 삭제** — 첨부된 사진 제거

### 위치 검색

- **SKU 위치 찾기** — 특정 제품이 등록된 셀 위치 반환

### 웹 에디터 호환

- **도면 JSON 조회** — 새 테이블에서 기존 형태의 JSON으로 조립하여 반환
- **도면 JSON 저장** — JSON을 받아 새 테이블에 분해 저장
