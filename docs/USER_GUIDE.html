v5.4.7
1. 첫 화면 둘러보기

PDA 전원을 켜면 앱이 자동 실행됩니다. 화면 구성을 먼저 익혀두세요.

┌────────────────────────────────────┐ │ [검색창] [🔍] [⚙] │ ← 우측 위 ⚙ = 설정 ├────────────────────────────────────┤ │ │ │ 📐 창고 도면 (격자) │ ← 중앙: 도면 │ 색상별 셀 표시 │ │ │ ├────────────────────────────────────┤ │ [🔄 배치] [🗂 박스] │ ← 하단 네비 └────────────────────────────────────┘
  • 검색창: 바코드/상품명 입력
  • 톱니바퀴(⚙): 서버 주소·연결 설정
  • 중앙 도면: 창고 셀(선반) 위치 시각화
  • 배치: 상품을 셀에 배치하는 화면
  • 박스: 박스 등록·관리 화면
2. 서버 주소 설정 (네트워크 변경 시 필수)

2.1 먼저 PDA가 어떤 WiFi에 연결됐는지 확인

  1. PDA 홈 화면 위쪽 상태바를 아래로 두 번 쓸어내림 (퀵 설정 펼침)
  2. WiFi 아이콘 길게 누름 → WiFi 설정 화면
  3. 화면 맨 위 "연결됨" 옆에 표시된 WiFi 이름 확인
    예: WAREHOUSE_5G, Office_WiFi 등

2.2 WiFi에 따라 입력할 서버 주소

연결된 WiFi 서버 주소
창고 내부 WiFi http://100.125.17.60:8000
사무실 WiFi http://100.125.17.60:8000
외부망 / 핫스팟 접속 불가 — 창고 WiFi로 변경
💡 미니PC 서버는 Tailscale이라는 보안 네트워크로 운영. 외부에서는 접속 불가, 회사 WiFi에 붙어 있어야 합니다.

2.3 서버 주소 입력 절차

  1. 메인 우측 상단 톱니바퀴(⚙) 탭
  2. 서버 URL 입력칸 탭하고 위 표 주소 그대로 입력
  3. 저장 탭
  4. 연결 테스트 탭 → "연결 성공" 토스트 확인
  5. 뒤로가기로 메인 복귀

2.4 "연결 실패" 토스트가 뜨면

  • 주소에 오타 없는지 확인 (http://, :8000)
  • WiFi 다시 한 번 확인
  • 그래도 실패 → 13장 응급 복구 참고
3. 바코드 스캔으로 상품 찾기

3.1 스캔 절차

  1. 바코드(검은 줄무늬)를 PDA 정면 위쪽 레이저 창을 향하게
  2. PDA 양옆/윗면의 노란색 트리거 버튼 누름
  3. 빨간 빛 + "삑" 소리 → 0.5초 내 상품 정보 표시

3.2 스캔 결과

  • 메인 하단에 카드 표시 (상품명/카테고리/위치/이미지)
  • 카드 탭하면 상세 페이지로 이동

3.3 스캔이 안 될 때

증상 해결
빨간 빛 안 나옴 트리거 버튼 끝까지 눌렀는지 확인
빛은 있는데 인식 안 됨 거리 10~20cm 유지, 너무 가까우면 안 됨
계속 실패 바코드 닦기 → 검색창에 직접 입력
모든 게 안 됨 PDA 재부팅 (전원 길게 → 다시 시작)
4. 텍스트 검색

4.1 입력 가능한 것

  • 상품명 일부: "케이스", "충전기" 같은 단어
  • 바코드 번호: 8~13자리 숫자
  • SKU 번호: 쿠팡 SKU 번호

4.2 절차

  1. 메인 검색창 탭 → 키보드 올라옴
  2. 검색어 입력 → 키보드의 🔍 돋보기 탭
  3. 결과 카드 목록에서 원하는 상품 탭 → 상세
💡 결과가 너무 많으면 더 구체적으로: "케이스" → "아이폰 15 케이스"
바코드/SKU로 검색하면 정확히 1건만 나옴
5. 상품 상세 페이지
┌────────────────────────────────────┐ │ [←] 상품명 │ ← 좌측 위 ← = 뒤로 │ SKU / 카테고리 / 바코드 │ ├────────────────────────────────────┤ │ │ │ 상품 이미지 │ ← 탭 = 썸네일↔실사 │ │ ├────────────────────────────────────┤ │ [📐 도면] [🛒 발주] [🏷 라벨] │ ← 하단 네비 └────────────────────────────────────┘
영역 동작
상단 상품명 정보 표시만
바코드 텍스트 길게 누르면 클립보드 복사
중앙 이미지 탭하면 썸네일 ↔ 실사 사진 전환
도면 창고 도면 → 이 상품 위치 노란색 강조
발주 구글시트 자동 등록 → 6장
라벨 바코드 라벨 인쇄 → 7장
6. 발주 (구글시트 자동 등록)

6.1 절차

  1. 상품 상세에서 🛒 발주 탭
  2. 수량 입력 다이얼로그가 뜸
  3. 빠른 선택 (1/10/50/100) 또는 직접 입력 (1~999)
  4. 발주 탭 → "발주 추가 완료" 토스트 (1~2초)
💡 구글시트에 자동으로 한 줄 추가됩니다.
날짜시간 / SKU / 상품명 / 수량
시트는 PC 브라우저로 직접 열어서 확인 (PDA 앱에는 발주 내역 조회 기능 없음)

6.2 발주 실패 메시지별 대응

메시지 대응
발주 실패: 시트 연동 실패 담당자 호출
발주 실패 (HTTP 500) 1분 후 재시도 → 안 되면 담당자
연결 오류 WiFi 확인 후 재시도
토스트 안 뜸 앱 강제종료 → 재실행
7. 라벨 인쇄 (중요!)

7.1 절차

  1. 상품 상세에서 🏷 라벨 탭
  2. 수량 입력 (1/5/10/100 또는 직접 1~100)
  3. 인쇄 탭
  4. 물류PC의 프린터에서 라벨 출력
  5. "N장 인쇄 완료" 토스트가 뜨면 성공

7.2 인쇄 실패 메시지별 대응

메시지 대응
물류PC 응답 없음 아래 7.3 절차 직접 수행
프린터 오프라인 프린터 전원/USB 케이블 확인
상품명과 라벨이 다름 담당자 호출
라벨 템플릿 없음 담당자 호출

7.3 "물류PC 응답 없음" 직접 살리기 (3분)

물류PC에 가서 다음 절차를 따라하세요.

  1. 물류PC 모니터 앞에 앉아서 마우스 흔들어 화면 깨우기
  2. 키보드 Win 키 → cmd 입력 → 엔터
    (검은 창 = 명령 프롬프트가 뜸)
  3. 검은 창에 아래 한 줄을 그대로 입력하고 엔터: 
    pythonw C:\print_agent.py
  4. 별다른 메시지 없이 명령 끝남 (정상)
  5. 검은 창은 닫지 말고 최소화 (창 닫으면 같이 꺼짐)
  6. PDA에서 라벨 인쇄 다시 시도
"pythonw 인식되지 않습니다" 오류 시
아래 두 가지 중 하나 시도: 
C:\Python311\pythonw.exe C:\print_agent.py
python C:\print_agent.py
그래도 안 되면 담당자 호출

7.4 PC 재부팅 후에도 자동 실행되게 하려면

명령 프롬프트에서 (한 번만):

reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v PrintAgent /t REG_SZ /d "pythonw.exe C:\print_agent.py" /f
8. 창고 도면

8.1 도면 여는 방법

방법 절차
메인 화면 항상 도면이 보임
상품 상세 📐 도면 버튼 → 상품 위치 노란색 강조

8.2 색상 의미

색상 의미
🟡 노란색 현재 검색 중인 상품 위치 (깜빡임)
🟢 녹색 상품이 보관된 일반 셀
⬜ 회색 비어있는 셀
🔲 검정 통로
🟦 파란색 테이블 / 작업대
🟥 빨간색 물류PC 위치

8.3 도면 조작

  • 두 손가락으로 벌리기: 확대 (핀치줌)
  • 두 손가락으로 좁히기: 축소
  • 셀 탭: 그 셀의 상세 페이지로 이동
9. 셀 상세 / 사진 변경

9.1 셀 상세 화면

┌────────────────────────────────────┐ │ [←] │ │ │ │ 📷 셀 사진 (풀스크린) │ ← 핀치줌 │ │ │ [A구역 1-1] │ ← 셀 위치 라벨 ├────────────────────────────────────┤ │ [📐 도면] [라벨] [✏️ 편집] │ └────────────────────────────────────┘
  • 한 셀에 사진 한 장만 등록됨
  • 하단 라벨: 어느 구역의 어느 셀인지 (예: "A구역 1-1")

9.2 셀 사진 변경 절차

  1. 셀 상세에서 ✏️ 편집 탭
  2. "새 사진을 업로드하시겠습니까?" → 확인
  3. 카메라 / 갤러리 선택
  4. 촬영/선택 후 자동 업로드
  5. "업로드 완료" 토스트 → 즉시 갱신

9.3 사진 촬영 팁

  • 셀 안 상품이 한 화면에 다 보이도록 조금 떨어져서 촬영
  • 어두우면 PDA 플래시 켜기 (카메라 좌측 ⚡)
  • 셀 라벨도 같이 보이게 찍으면 다음에 찾기 쉬움

9.4 사진 업로드 실패 시

증상 대응
"업로드 실패" 토스트 편집 → 사진 변경 다시 시도
계속 실패 WiFi 신호 확인 (우측 위 막대)
사진이 90도 회전됨 알려진 이슈, 갤러리에서 회전 후 갤러리로 업로드
모두 실패 담당자 호출
10. 박스(QR) 스캔

10.1 스캔 절차

  1. 외박스의 QR 코드를 PDA 정면을 향하게
  2. 트리거 버튼 → "삑" 소리
  3. 박스 상세 페이지로 자동 이동

또는 검색창에 박스 번호 직접 입력: BOX-1234

10.2 박스 상세

  • 상단: 박스에 담긴 상품명, 보관 위치
  • 중앙: 박스 사진 + 도면 (위치 노란색 강조)
  • 하단 외부 링크: 네이버 / 쿠팡 / 1688 / FLOW (해당 링크 있을 때만 활성)
11. 앱 업데이트

11.1 자동 업데이트

  1. 앱 실행 시 새 버전이 있으면 다이얼로그 자동 표시
  2. 업데이트 탭 → 자동 다운로드 (10~30초)
  3. "설치하시겠습니까?" → 설치
  4. 앱 자동 재시작

11.2 알림이 안 뜰 때

  1. 최근 앱 (▢) → 위로 쓸기 → 다시 실행
  2. 안 뜨면 → 이미 최신이거나 서버 연결 안 됨

11.3 현재 버전 확인

설정 (톱니바퀴) 하단에 표시. 예: v5.4.7

12. 자주 묻는 질문 (FAQ)
Q1. 바코드 스캔이 안 돼요
트리거 끝까지 → 빨간 빛 확인 → 거리 10~20cm → 바코드 청소 → 검색창 직접 입력으로 우회
Q2. "서버 연결 실패" 떠요
상태바 아래로 쓸기 → WiFi 확인 → 창고 WiFi 아니면 변경 → 서버 URL 재확인 (http://100.125.17.60:8000) → 그래도 실패 시 담당자
Q3. 상품을 찾을 수 없다고 떠요
다른 검색어 (상품명/SKU) → 신규 상품이면 담당자에게 등록 요청
Q4. 라벨이 안 찍혀요 — "물류PC 응답 없음"
7.3 절차 그대로 (cmd 열고 pythonw C:\print_agent.py)
Q5. 발주 시트에 안 들어가요
"발주 추가 완료" 토스트 확인 → 시트 새로고침 → 1~2초 지연 가능 → 시트 연동 실패 메시지 시 담당자
Q6. 셀 사진이 옛날 거로 보여요
다른 PDA에서 변경된 경우 최대 1분 지연 → 뒤로가기 → 다시 진입하면 즉시 갱신
Q7. 사진이 90도 돌아서 업로드돼요
알려진 이슈, 다음 업데이트에서 수정 예정. 임시: 갤러리에서 회전시켜 저장 후 갤러리로 업로드
Q8. 앱이 멈추거나 느려요
최근 앱 (▢) 위로 쓸기 → 재실행 → PDA 재부팅 → 새 업데이트 확인
Q9. PDA 화면이 너무 빨리 꺼져요
PDA 시스템 설정 → 디스플레이 → 화면 자동 꺼짐 시간 늘리기 (앱에서 제어 안 함)
Q10. 다른 사람도 같은 PDA 써도 되나요?
네, 누구나 사용 가능. 단 발주/사진 변경한 사람은 기록 안 됨
Q11. WiFi가 자꾸 끊겨요
신호 약한 곳일 수 있음 (구역 끝, 컨테이너 안 등) → 신호 좋은 곳으로 이동
Q12. 도면이 안 보이거나 깨져요
뒤로가기 → 다시 진입 → 그래도 안 되면 앱 강제종료 → 재실행 → 도면 자체 문제면 담당자에게 도면 재설정 요청
13. ⚡ 응급 복구

13.1 PDA 한 대만 안 됨

  1. 앱 강제종료 → 재실행 (▢ → 위로 쓸기)
  2. WiFi 끄기 → 5초 → 다시 켜기
  3. PDA 재부팅 (전원 길게 → 다시 시작)
  4. 다른 PDA로 동일 작업 시도 → 다른 PDA에서 정상이면 이 PDA 문제 → 담당자에게 PDA 점검

13.2 모든 PDA가 안 됨 (서버 다운 가능성)

  1. 다른 직원 PDA도 확인 (모두 같은 증상이면 서버 문제)
  2. 미니PC 모니터 확인 — 빨간 에러? 인터넷 케이블 LED?
  3. 미니PC 재부팅 (담당자 동의 후) → 시작 → 전원 → 다시 시작 → 2~3분 후 PDA에서 시도
  4. 그래도 안 되면 담당자 즉시 호출

13.3 라벨만 안 찍힘 (스캔/검색은 정상)

  1. 물류PC 켜져 있는지
  2. 프린터 전원/USB 확인
  3. 7.3 절차로 인쇄 에이전트 실행
  4. PDA에서 라벨 다시 인쇄
  5. 그래도 안 되면 라벨 템플릿(*.btw) 누락 → 담당자

13.4 발주만 안 됨

  1. WiFi 신호 확인
  2. 1분 후 재시도 (구글시트 일시 지연)
  3. PC 브라우저로 시트 직접 열어보기 — 안 열리면 구글 측 문제
  4. 그래도 안 되면 담당자

13.5 도면이 이상하게 보임

  1. 앱 강제종료 → 재실행
  2. 톱니바퀴 → 메인 다시 진입
  3. 그래도 이상하면 도면 데이터 손상 가능 → 담당자
